|---|---|---|---|---|---|

| 联合类型 (A/B) | ⬜ | ⬜ | ✅ | 🔴 | 需密封类模拟 |
| 交叉类型 (A&B) | ⬜ | ⬜ | ✅ | 🔴 | JVM 多接口继承可部分模拟 |
| Nothing/Never 底部类型 | ⬜ | ✅ | ✅ | 🟢 | 表示永不返回 (如 throw 表达式) |
| Unit/Void 类型 | ✅ | ✅ | ✅ | 🟢 | void 返回类型 |

| 原始类型 (primitives) | ✅ | ⬜ | ⬜ | 🟢 | int8/16/32/64, float32/64, string, boolean |
| 无符号整数 | ⬜ | 🔶 | ⬜ | 🔴 | JVM 无原生无符号类型 |
| 数组类型 | ✅ | ✅ | ✅ | 🔴 | 需设计 Mammoth-PHP 数组语义 |

| 值类型/内联类 | 🔶 | 🔶 | ⬜ | 🔴 | 依赖 Valhalla |
| 运行时类型信息保留 | 🔶 | 🔶 | ✅ | 🔴 | 泛型擦除导致运行时类型丢失 |

### ⚫ 无法实现（JVM 平台根本限制）

| 特性 | 原因 |
|---|---|
| `eval()` | JVM 不支持运行时编译 + 加载执行，需内嵌完整编译器 + 类加载隔离 |
| `$$var` 可变变量 | 变量名在字节码中被抹除，无运行时符号表查询 |
| 运行时动态属性添加 | JVM 类结构加载后固定 |
| `__call` / `__get` / `__set` 魔术方法 | 字节码成员访问是直接指令 (invokevirtual/getfield)，无法运行时拦截 |
| `__destruct` 确定性析构 | GC 时机不可预测，Cleaner API 语义不等价 |
| 动态类型 / 渐进类型 | 编译期需要确定所有类型，bytecode 无法运行时改变类型派发 |
| `include` / `require` 动态加载 | JVM 类加载有严格生命周期，无法在方法内动态引入执行外部文件 |
| 字面量作为类型 | JVM 类型系统不支持值作为类型 |
